Offensive Security Introduction

Web Architecture

  • Web 1.0: Static, no interaction.
  • Web 2.0: Dynamic, realtime interaction.

Web App Components

  • Front End: HTML (Structure), CSS (Style), JS (Script/Logic).
  • Back End: Servers (Apache/Nginx), Databases (SQL/NoSQL), Frameworks (Laravel, Django, Express).

Vulnerabilities

1. Injection Attacks
  • SQL Injection: Malicious SQL queries.
  • Command Injection: Executing OS commands.
  • HTML Injection: Unfiltered user input displayed on page.
2. XSS (Cross-Site Scripting)

Injecting JS code on client-side.

  • Reflected, Stored, DOM-based.
  • Prevention: Sanitization and Validation.
3. Broken Authentication

Bypassing login or gaining admin privileges incorrectly.

Databases

  • Relational (SQL): Tables/Rows (MySQL, PostgreSQL).
  • Non-Relational (NoSQL): Key-Value/Documents (MongoDB, DynamoDB).
