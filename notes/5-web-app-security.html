What is it?

Protect websites and apps from unauthorized access, changes, or destruction.

History of the Web

  • ARPANET (1969): First network connecting researchers, focused on sharing info
  • World Wide Web (1989): Tim Berners-Lee invented it to make info easier to access
  • Early Web (Web 1.0): Static pages only
  • Modern Web: Interactive, dynamic, works across devices using HTML5, CSS3, JavaScript

Why Security Matters

  • Confidentiality: Only the right people see data (ex: online banking passwords)
  • Integrity: Info stays accurate (ex: money transfers logged and verified)
  • Availability: Systems work even if there are failures (ex: backups, redundant servers)
Example Case: Ransomware Attack
Blackcat group attacked Change Healthcare.
Weakness: no multi-factor authentication on a Citrix server.
Result: $3.09 billion loss.

How to Stay Safe

  • Use MFA for important systems
  • Train employees on security
  • Run vulnerability checks and penetration tests
  • Keep software updated

Anatomy of a URL

  1. Protocol: How browser talks to server (http, https, ftp)
  2. Hostname: Server name (domain like google.com)
  3. Path: Location of a file on the server
  4. Port (optional): Specific “door” on server
  5. Query Parameters (optional): Extra info like search filters
  6. Fragment/Anchor (optional): Jump to a section in a page

Web Security Challenges

  • Human Factor: Users often don’t understand security
  • Complex Architecture: Web apps are complicated with client-server interactions
  • OWASP: Guides for web security (Top 10 list of risks)
