CWE-35: Path Traversal

A vulnerability that lets attackers access files or directories outside intended areas.

How it works

Attackers use dot-dot-slash notation to move up directory trees.

../ (Unix) OR ..\ (Windows)

Attack Techniques

  • Simple: ../../etc/passwd
  • Percent encoding: %2e%2e%2f
  • Double encoding: %252e%252e%252f
  • NULL bytes: filename.php%00.jpg

What can happen

  • Read sensitive files (like /etc/passwd)
  • Modify or delete important files
  • Gather system info for further attacks

Prevention

  • Validate user input with allowlists.
  • Avoid using raw user input in file paths.
  • Use built-in path normalization functions.
  • Apply least privilege access controls.

Testing Tools

Burp Suite, OWASP ZAP, Nikto, DotDotPwn, Metasploit, DirBuster.
