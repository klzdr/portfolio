What is SQL Injection?

Happens when software builds SQL queries using untrusted user input without cleaning it. Attackers can insert SQL code to manipulate the database.

Quick SQL Refresher

  • Retrieve data (SELECT)
  • Insert data (INSERT)
  • Update data (UPDATE)
  • Delete data (DELETE)

How injection works

User input is combined with SQL commands directly.

// Intended:
SELECT * FROM users WHERE username = 'USER' AND password = 'PASS';

// Malicious input: ' OR '1'='1
SELECT * FROM users WHERE username = '' OR '1'='1' AND password = 'PASS';

This result always evaluates true, potentially logging in the attacker.

How to prevent it

  1. Prepared Statements / Parameterized Queries: SQL structure compiled first, then input added as data, not code.
  2. Input validation: Check type, length, format, allowed characters.
  3. Escaping input: Add slashes before special symbols (less secure).
  4. Least Privilege: Give database accounts minimum access needed.
