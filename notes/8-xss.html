CWE-79: Cross-Site Scripting (XSS)

XSS allows attackers to inject scripts into web apps, affecting users. It bypasses the browser’s same-origin policy to run malicious scripts.

How XSS works

  1. Attacker injects malicious JavaScript into a page.
  2. Victim’s browser executes it.
  3. Example: URL parameter updates a page without proper input handling.

Types of XSS

  • Reflected (Non-persistent): Script is in the page response (not stored). Only affects users who click a crafted link.
  • Stored (Persistent): Script stored on server (database, comments). Affects all users who view the page.
  • DOM-Based: Client-side only. Attacker changes the DOM via JavaScript. Payload may never touch the server.
  • Universal XSS (uXSS): Exploits browser or extension flaws.
  • Self-XSS: Social engineering trick to get users to run scripts in their own console.

Consequences

  • Stealing cookies or sessions (Account Takeover)
  • Data theft
  • Website defacement
  • Malware delivery

Prevention

  • Validate and Sanitize all input.
  • Encode output depending on context.
  • Use HTTPOnly cookies.
  • Implement Content Security Policy (CSP).
